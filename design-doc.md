## Problem Statement

KP has 37 GitHub repos spanning legal AI agents, investment tools, Claude Code skills, Obsidian utilities, content dashboards, and creative projects — but no curated presentation layer. Visitors to the GitHub profile see an undifferentiated list. The showcase page will be a single-page cinematic portfolio that tells the story of KP as a builder, with curated repos organized by category, live GitHub stats, and Awwwards-level visual craft.

## What Makes This Cool

This isn't just a repo list — it's an immersive scroll experience. Think Apple product page meets developer portfolio: a full-viewport hero that introduces KP's work, scroll-triggered section reveals, buttery smooth Lenis scrolling, animated stat counters, glassmorphic cards with hover effects, and a cinematic dark luxury aesthetic with subtle purple-blue gradient accents. The kind of page where someone lands and thinks "this person has serious taste."

2. **Stats Bar** (scroll-triggered)
   - Animated number counters: Repos, Stars, Categories, Languages
   - Numbers count up from 0 when scrolling into view

3. **Category Navigation** (sticky, `z-index: 50`)
   - Pill-shaped filter buttons
   - Smooth GSAP transitions when filtering
   - Categories: Legal AI, Investment, Claude Code, Obsidian, Content & Web, Fun & Creative
   - Active state with subtle glow

4. **Featured Repo** (full-width hero card)
   - Highlight the best repo (e.g., contract-review-agent with 20 stars)
   - Two-column: description + visual (screenshot/code snippet/animation)
   - Scroll-triggered reveal with parallax

5. **Repo Grid** (2-column CSS Grid, equal-height rows)
   - Cards with: category tag, repo name, description, language dot, star count, last updated
   - Hover: subtle lift, gradient top-border reveal, background brighten

Clicking a category pill: (1) non-matching cards fade to 0 over 200ms then `display: none`, (2) matching cards stagger-fade-in from opacity 0 → 1 over 300ms with `stagger: 0.06`. 6. **Footer**
   - GitHub / LinkedIn / other social links
   - "Stats powered by GitHub API" note
   - Subtle, minimal

### Visual Design System

- **Palette**: Deep black (#0a0a0f) background, warm white (#e8e4df) text, purple-blue gradient accents (#c9a0ff → #7eb8ff)
- **Typography**: Inter (self-hosted via `@font-face` from `/public/fonts/`; weights 200, 400, 500, 700). Fallback: `-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`. Korean text: `'Noto Sans KR'` loaded from Google Fonts (weights 400, 700)
- **Cards**: Near-transparent backgrounds (rgba white 2-4%), thin borders (rgba white 6-12%), 16px border radius
- **Hover effects**: Card lift (translateY -4px), gradient top-border fade-in, background brighten
- **Animations**: Ease out cubic-bezier(0.16, 1, 0.3, 1) for smooth, premium feel
- **Spacing**: Generous whitespace, 80-100px between sections

### Responsive Behavior

- **Breakpoints**: 1024px (tablet), 768px (mobile)
- **Grid**: 2-column → 1-column at 768px
- **Hero**: Typography scales down (56px → 36px → 28px), subtitle narrower
- **Category pills**: Horizontally scrollable row on mobile (no wrapping)
- **Featured card**: 2-column → stacked (visual below text) at 768px
- **Stats bar**: 4 across → 2x2 grid at 768px
- **Lenis**: Do NOT initialize Lenis on mobile/touch devices (detect via `matchMedia('(pointer: coarse)')` or `window.innerWidth < 1024`). Native scroll only. GSAP ScrollTrigger works with native scroll — no code changes needed
- **Touch**: No hover effects on touch devices; cards have tap-to-expand instead

### Accessibility

- **`prefers-reduced-motion`**: All GSAP animations and Lenis smooth scroll disabled; content appears immediately, no transitions
- **Keyboard navigation**: Category pills are `<button>` elements with visible focus rings; repo cards are `<a>` links wrapping the card
- **ARIA**: Category filter group uses `role="toolbar"` with `<button aria-pressed="true/false">` for each pill; stat counters use `aria-live="polite"` for screen readers
- **Color contrast**: All text meets WCAG AA (4.5:1 minimum). Card description text uses `rgba(255,255,255,0.55)` on #0a0a0f background (~5.0:1). Secondary metadata text at `rgba(255,255,255,0.52)` (~4.7:1)
- **Semantic HTML**: `<header>`, `<main>`, `<nav>`, `<section>`, `<footer>` throughout

### SEO & Social Sharing

- `<title>`: "KP's Open Source Showcase — AI Agents, Legal Tech, Developer Tools"
- `<meta name="description">`: Curated portfolio summary
- Open Graph: `og:title`, `og:description`, `og:image` (og-image.png — 1200x630 branded card), `og:url`
- Twitter Card: `twitter:card=summary_large_image`, `twitter:title`, `twitter:description`, `twitter:image`
- `<html lang="ko">` (default), toggled to `en` via i18n switch

### Bilingual (i18n) Specification

- **UI**: Pill toggle `position: fixed` top-right corner ("KO / EN"), `z-index: 100` (above all content including sticky category nav). Persisted in `localStorage`
- **Default**: Korean (ko)
- **Scope**: All translatable content — hero tagline, subtitle, section labels, repo descriptions, UI labels, footer text. Repo names stay as-is (they're code identifiers)
- **Mechanism**: Each translatable string has `data-i18n="key"` attribute; `i18n.js` swaps `textContent` from a translations map. `<html lang>` attribute updates accordingly
- **Transition**: Text fades out (150ms) → swap → fades in (150ms) via CSS opacity

### Data Architecture

```javascript
// repos.js — curated repo data with categories
export const repos = [
  {
    name: 'contract-review-agent',
    category: 'Legal AI',
    featured: true,
    description: { ko: '...', en: '...' },
    // GitHub API fills: stars, forks, language, updated_at
  },
  // ... curated selection
];
```

- **Primary data source**: Build-time fetch via GitHub Actions. A `fetch-stats.js` script runs during CI using the auto-provided `GITHUB_TOKEN` (read access to public repos, no setup needed), calls the GitHub REST API for each curated repo (stars, forks, language, updated_at), and writes results to `src/repo-stats.json`. This file is imported by `repos.js` and baked into the bundle.
- **Build failure fallback**: If `fetch-stats.js` fails (API down, network error), the build uses the last committed `repo-stats.json` and logs a warning. The build does not fail — stale stats are better than no deploy.
- **Freshness**: GitHub Actions scheduled daily (`cron: '0 6 * * *'`) to rebuild and redeploy with fresh stats. Manual push also triggers a rebuild.
- **Optional client-side refresh**: On page load, fetch ONLY the featured repo's stats (1 API call, not all repos) to show the most current star count on the hero card. On failure, silently use build-time data. Wrap `sessionStorage` access in try/catch (fallback to in-memory variable for private browsing).
- **Loading state**: Skeleton shimmer effect on the featured repo's star count while the single client-side fetch runs (< 500ms). No layout shift — all other stats display build-time data immediately.
- Language colors from GitHub's linguist colors (hardcoded map for the languages used: Python, TypeScript, HTML, JavaScript)
